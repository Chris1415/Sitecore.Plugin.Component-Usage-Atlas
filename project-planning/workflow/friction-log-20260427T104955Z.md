### 2026-04-27 â€” T008 / M1 verification â€” `notFound()` returns HTTP 200, not 404

ADR-0014 mandates "root route â†’ `notFound()`" and T008 expected output states "`http://localhost:3000/` returns Next.js 404." Verified in browser: `notFound()` IS firing server-side (RSC payload contains `NEXT_HTTP_ERROR_FALLBACK;404` and the standard `"404: This page could not be found"` markup). However:

- HTTP wire-level status is **200**, not 404. Next 16 App Router renders `notFound()` content **inside the root layout**, which means `<MarketplaceProvider>` wraps the rendered 404. While the provider is in `loading === true` state (the default outside the portal iframe), its loader (`<div>Attempting to connect to Sitecore Marketplace...</div>`) renders in place of children â€” so the user sees the loader, not the 404 page.
- A direct hit to `/_not-found` returns HTTP 404 cleanly (the standalone not-found route bypasses the layout), confirming Next is correctly configured.
- A real undefined route (`/does-not-exist`) also returns HTTP 404 normally.

Practically harmless â€” editors never visit `/` in the portal (they hit `/widget` or `/panel` extension routes), and a developer doing local smoke-test sees the same loader on `/widget` and `/panel` outside the iframe, so `/`'s loader is consistent. ADR-0014's intent ("root unreachable") is met at the application-semantics level even if the HTTP code isn't 404.

**Skill patch / ADR amendment suggestion:** ADR-0014 should acknowledge the layout-precedence behavior in Next 16 App Router. Either (a) amend the ADR to say "root renders the provider's standby state in standalone-localhost; portal-iframe behavior is the only one that matters" or (b) consider moving `<MarketplaceProvider>` from `app/layout.tsx` to nested `app/widget/layout.tsx` + `app/panel/layout.tsx` so root is provider-free and 404s cleanly. Option (b) contradicts T008's "Do NOT remove the root provider" guard and may break the panelâ†”widget switch UX, so option (a) is the safer call. Recommended: revisit at M6 / `/architect` review pass.
