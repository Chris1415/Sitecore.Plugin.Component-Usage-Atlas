## 1. Summary

PRD-001 (Atlas Snapshot Export) ships a portable-snapshot egress on top of the PRD-000 live in-memory atlas. Editors can take JSON, CSV, or HTML snapshots out of the iframe via three first-class actions per surface — Save, Open in new tab, Copy to clipboard — without breaking any of the architectural posture from PRD-000 (no backend, no persistence, no new SDK calls, no new extension points). The original spec assumed a single Download button; the T001 verification spike on 2026-05-04 confirmed downloads are silent-blocked in the Marketplace iframe sandbox (host omits `allow-downloads` — a known platform-level limitation also documented in the sibling Pageshot product). The plan forked mid-implementation to the proven pageshot three-action pattern (ADR-0021 supersedes-in-part ADR-0017). Save renders disabled today and stays in code as future-proof; Open + Copy ship as primary user-visible egress. Status is `shipped_with_caveats` because four manual / measurement smoke gates are still `pending` — host-frame visual smoke, HTML print preview, live walkthrough, and bundle-cap NFR-1.4 verification — none of which are code defects.

## 4. Known Limitations

- **Save renders disabled in the current Marketplace iframe sandbox.** Tooltip: *"Save will become available as soon as the Sitecore Cloud Portal grants the iframe download permission. Until then, use Open or Copy."* The hook is fully implemented; one guard flip at the surface level enables it the moment Sitecore platform adds `allow-downloads`.
- **`useOpenExport` `'blocked'` detection is imperfect.** `noopener,noreferrer` causes browsers to return null from `window.open` even when the popup actually opens (browser quirk surfaced live by the user during smoke). The `'blocked'` status auto-reverts to `'idle'` after 3.5 s so a false positive doesn't permanently disable Open. A real popup-block event is briefly visible; a false positive disappears.
- **Clipboard HTML mode pastes formatted markup into rich-text targets.** By design per ADR-0021 — receivers pasting into Slack / email get formatted output; pasting into a code editor gets raw HTML. Documented; A/B candidate if editor friction signal surfaces.
- **No PDF library shipped (ADR-0018).** Editors who want a PDF use the browser's print dialog on the HTML output (Open in new tab → Ctrl+P → Save as PDF). Adds one editor-side step; bundle stays small.
- **Bundle delta exceeds DoD-5 cap by ~1.9× (estimated).** See D-4 above.
- **POC frames are stale on the action-cluster aspect.** See D-5 above.
