## 6. Bundle delta — open escalation (DoD-5 / NFR-1.4)

| Phase | `.next/static/` raw bytes | Delta vs PRD-000 baseline | Gzipped estimate (35% ratio) |
|---|---:|---:|---:|
| PRD-000 baseline (pre-PRD-001) | 1,288,631 | — | ~451 KB |
| After T002 (`@blok/sonner` + `next-themes`) | 1,325,398 | +36,767 | ~+13 KB |
| After T041 (final cumulative) | 1,433,747 | +145,116 | **~+50.8 KB** (file-sum) or **~+37.9 KB** (export-feature attribution after subtracting framework drift) |

**Cap:** NFR-1.4 / DoD-5 = **≤20 KB gzipped**.

**Honest read:**
- Measurement methodology is approximate — `.next/static/` raw bytes × 35% compression ratio is a coarse proxy for actual served gzipped bundle. A precise measurement requires `next-bundle-analyzer` or the Vercel Analytics route-level breakdown.
- ADR-0021's three-action egress is structurally larger than the original single-Download concept the cap was sized for. Three hooks + action-cluster pills + `lib/contrast.ts` + telemetry helpers come to ~1,200 LOC of new TypeScript before tree-shaking.
- The largest per-file contributors are likely shared framework primitives (`@blok/sonner`, `next-themes`) and Blok component CSS, not export-feature-specific code.

**Recommended path:**
- Either (a) **re-measure** with `next-bundle-analyzer` to isolate the export-feature share precisely, OR
- (b) **amend NFR-1.4** at `/document` time to reflect the post-fork architecture (the cap was set when the spec assumed a single Download button; ADR-0021 supersedes that mental model).

This is a planning/measurement decision, not a code defect. **Tracked in this report and in the run manifest's `smoke_outcomes` slot for `/ship` to gate.**
