> **Reading note.** This is variant **1 of 3** for the Component Usage Atlas UI. The named direction is **"Atlas Cartographer"** ‚Äî the tenant is treated as a map, the widget is the world index, the panel is "you are here" with mini-radar showing fan-out. All component names below are **real Blok primitives or composite blocks** (`@blok/*`). All colors and spacing reference **Blok semantic tokens** ‚Äî never raw hex, never invented token names. Where a needed primitive does not exist in the Blok registry, the gap is called out and a composition strategy is provided.

## 1. Design context & assumptions

### 1.1 ‚Äî Audience

- **Primary:** content editors on the Sitecore Pages canvas, mid-task, who are about to publish/delete/modify and want to know what else is affected. They live inside Pages, recognize renderings by display name (not template ID), and care about pages ‚Äî not items (per PRD ¬ß 4 / A-PERS-1, A-PAIN-5).
- **Secondary (incidental):** information architects auditing component usage; developers preparing migrations. v1 does not optimize for them but the dense, table-first widget gives them what they need.
- **Anti-persona:** the text-only editor whose work is limited to copy edits ‚Äî Atlas's value is invisible to them; the widget is opt-in at install time and the panel is collapsed by default.

### 1.2 ‚Äî Goals and success criteria

| Goal (PRD) | What this design must support |
|------------|--------------------------------|
| G1 ‚Äî component-centric answer | Widget primary action is **search-then-scan-the-table**; row click opens a drawer with cross-tenant page list, with rendering-direct vs via-datasource sections kept visually distinct. |
| G2 ‚Äî page-centric answer | Panel reads "you are here" first; for every rendering on the active page, surface a **+N other pages** counter as a map-grid coordinate label. |
| G3 ‚Äî datasource impact | A separate, equally-weighted **Datasource impact** group below the rendering list, with click-through to the same drawer pattern. The mandatory **`‚ìò Direct bindings only`** affordance (ADR-0006 / FR-8.3) is non-dismissible and visible from both surfaces. |
| G4 ‚Äî bearable wait | A single branded "surveying" loading visualization (ADR-0011) ‚Äî phased ring/pulse traces from sites ‚Üí pages ‚Üí components, leaving a faint trail. Cancelable, with phase + count + elapsed time. 60fps via CSS animations only. |
| G5 ‚Äî zero-infra | Design must not imply any persisted state. "Last scanned" timestamp is the only memory cue; **Refresh atlas** is always reachable. |

Success per PRD ¬ß 3:
- **S2 ‚â§ 2s warm** ‚Äî the table must render from cached state without a perceptible spinner; use `@blok/skeleton` only for the brief idle‚Üícompleted transition.
- **S3 ‚â§ 30s cold** ‚Äî the loading visualization must look intentional, not broken, for at least 30s on a 1k-page tenant.
- **S6 "felt fast" ‚â• 70%** ‚Äî the loading visualization is the load-bearing aesthetic decision for this metric.
- **DoD-6** ‚Äî branded loading UX 60fps on a typical editor laptop.

### 1.3 ‚Äî Constraints

**Brand / design system (mandatory):**
- All UI is **Blok** (Sitecore design system) per ADR-0009. Components are added via `npx shadcn@latest add @blok/<name>`. The registry stylesheet (`theme.json`) supplies semantic tokens.
- **No** hand-picked hex. **No** invented `--blok-*` token names. **No** Google Fonts or custom font families ‚Äî the Blok typography stack (shipped by `theme.json`) is the only typography source.
- **Light + dark mode parity** is automatic via Blok semantic tokens; both modes inherit from the theme. No custom palette branch.

**Architecture-locked (cannot revisit at design time):**
- **Two iframe entries** ‚Äî Widget at `app/widget/page.tsx` (Cloud Portal Route URL `/widget`), Panel at `app/panel/page.tsx` (Route URL `/panel`). Same shared atlas singleton; different surfaces. (ADR-0004, architecture ¬ß 2.1)
- **Loading visualization is a branded animation** in v1. **No** generative-art bloom, **no** mini-game (ADR-0011 / IS-7 / OS-12). The "Distract me" tri-state from PRD AC-4.3 is **removed** in v1 ‚Äî there is only one mode.
- **Session-scoped cache** (browser-tab lifetime) ‚Äî surfaces show "Last scanned: HH:MM" with stale hint at >15 min (AC-5.1 / AC-5.2). No auto-refresh.
- **Rendering identity = definition ID; datasource identity = item ID** (ADR-0005). Display names are primary; item IDs only appear as tooltips, copyable badges, or click-through hrefs. Disambiguation suffix `¬∑ <last-7-of-id>` when display names collide (FR-9).
- **Direct datasource bindings only** (ADR-0006 / FR-8.3) ‚Äî the **`‚ìò Direct bindings only`** affordance is mandatory and non-dismissible wherever datasource counts appear.

**Iframe constraints (Marketplace):**
- Iframe widths in real tenants: 800px / 1024px / 1440px ranges (per `blok/theming.md` Marketplace rules). Test at all three.
- Shadows clip at iframe edges ‚Äî prefer `shadow-sm` / `shadow` from Blok scale; **no** large drop shadows.
- The widget surface gets the dashboard-block frame from the host; the panel surface is a narrow vertical column with no topbar (per `marketplace-integration.md`).

**Accessibility:**
- WCAG **AA** contrast across both modes (Blok tokens already meet AA).
- Keyboard navigation throughout ‚Äî `Tab` order, arrow-key row navigation in the table (NFR-4.1), `Esc` closes the drawer.
- `aria-live="polite"` on the loading visualization phase label and pages-completed counter (NFR-4.2 / ADR-0011).
- Color is **never** the only signal ‚Äî every status uses an icon + text label (NFR-4.3).

### 1.4 ‚Äî Reference / inspiration

The cartographic direction draws on three real-world visual idioms ‚Äî adapted, never copied:

- **Aviation sectional charts** (FAA / SkyVector) ‚Äî clean, technical, dense; thin line weights; precise typography; coordinate-pair labels (e.g., `40¬∞N 73¬∞W`). The Atlas's `+N` counters are styled as map-grid coordinates: monospace tabular figures, hairline border, neutral surface.
- **USGS topographic maps** ‚Äî isobar / contour lines used as separators. The "isobar separator" between the panel's rendering list and Datasource impact group is a single `@blok/separator` with `decorative` behavior; the cartographic feeling comes from spacing rhythm, not ornament.
- **Mapbox / Leaflet "you are here" pins** ‚Äî the panel's current-page header reads as a callout block (slightly heavier weight) over a neutral plate. No literal map pin icon ‚Äî too costume-y; we use a small `@blok/icon` (location indicator) at left of the page-name callout.
- **Index-of-charts / chart legends** ‚Äî the widget's table reads like a legend/index. Column headers are subdued; row text leads.

**Web search (light, summarized):** typical "data-dense map dashboards" (Datadog map view, Grafana world-map panel, Aviation chart browsers like ForeFlight) consistently use: low-saturation neutral surfaces, accent color reserved for "you" or "active scan," monospace numeric figures, hairline separators, and badge-style coordinate labels. The Atlas adapts those patterns without copying any specific UI.

**No external assets are pulled in.** No imported map tiles, no SVG borrowed from any chart provider ‚Äî the cartographic feel is a typographic + layout discipline, not a pictorial layer.

### 1.5 ‚Äî Answers to clarifying questions

These were not asked interactively ‚Äî variant brief locked them in:
- **Count and divergence:** 3 variants, radically different aesthetics. This is variant 1; variant 2 (TBD) and variant 3 (TBD) explore different metaphors.
- **Tone:** clean, technical, precise. Aviation-chart aesthetic on a Sitecore Cloud Portal canvas.
- **Loading visual:** "surveying pulse" ‚Äî a tiny moving point traces site ‚Üí page ‚Üí component, leaves a faint trail.
- **Counter style:** Blok `@blok/badge` rendered like a coordinate-pair label (monospace, tabular figures, hairline border).
- **Anchor pattern:** "You are here" callout with a slightly heavier type weight than the surrounding rendering list.

## 2. Design direction

### 2.1 ‚Äî Visual direction

**Named direction: "Atlas Cartographer."** The tenant is rendered as a map. The widget IS the world index ‚Äî a precise, dense legend of every rendering (continent) and how many pages (territories) it covers. The panel IS "you are here" ‚Äî the current page anchored in the legend, with mini-radar fan-out for each rendering pointing at "12 other pages share this."

Core visual moves:
1. **Surfaces stratify like a topographic map** ‚Äî three subtle neutral layers using only Blok semantic tokens: `bg-background` (the deepest), `bg-muted/40` (mid-relief, used for the search bar plate, the panel's anchor block, and table row hover), `bg-card` (raised plates for the freshness ribbon and the loading visualizer panel). No drop shadows on the panel; subtle `shadow-sm` only on the drawer when open.
2. **Hairline separators in place of heavy dividers** ‚Äî `@blok/separator` with `decorative` and the default border-token color. Wherever a horizontal rule would look heavy, we use Tailwind `border-b border-border/60` with the Blok semantic border token.
3. **Primary action color reserved** ‚Äî `colorScheme="primary"` on `@blok/badge` only fires for two states: (a) the "You are here" anchor in the panel, (b) the active scan progress ring/pulse. Everything else is neutral or muted. Editors should never feel a "wall of red dots" ‚Äî the primary color is the compass needle, and only one needle.
4. **Coordinate-pair counters** ‚Äî `+12 pages` is rendered as a `@blok/badge` with `variant="default"` + a leading north-east arrow icon (`@blok/icon`, name `arrow-up-right`), with `font-variant-numeric: tabular-nums` and `font-mono` (or the Blok mono fallback) for digits. The badge has a 1px border at `border-border/60` and rounded-sm corners ‚Äî reads as a chart label, not a notification dot.
5. **"Surveying" loading pulse** ‚Äî a tiny moving point that traces from a site icon (left) to a page icon (middle) to a component icon (right), leaving a faint trailing line that fades over ~600ms. The trail color is `--ring` (Blok focus ring color) at 40% opacity. Below: phase label, current/total, elapsed time, cancel button. Built entirely in CSS + a single SVG `<path>` per phase.
6. **Typography hierarchy bias** ‚Äî the page-context anchor in the panel and column headers in the widget table are the only places where "callout weight" appears. Every other label is body weight. This avoids the "everything bold" trap that sinks dashboards into noise.

Tone: **clean, technical, precise.** Like reading a sectional chart in flight prep ‚Äî every element is there because a question depends on it; nothing decorative.

### 3.3 ‚Äî Responsiveness

The two surfaces have different responsive concerns because the host frame controls their width.

**Widget (Dashboard Block).** Host gives the widget a width somewhere between **~720px** (narrow dashboard column) and **~1400px** (full-width tile). Heights vary; widget should fill the assigned height.

| Breakpoint (iframe width) | Behavior |
|---------------------------|----------|
| < 720px (theoretical, not expected) | Stack the freshness ribbon vertically; collapse "Last scanned" + scope picker into a single row with a `‚Ä¶` menu (`@blok/dropdown-menu`). Table reduces to two columns: rendering + pages count. |
| 720‚Äì1024px | Default. Search bar full-width; table shows all 4 columns. Drawer opens at `w-[480px]`. |
| 1024‚Äì1440px | Same as default; drawer expands to `w-[560px]`. |
| > 1440px | Same; drawer expands to `w-[640px]` for legibility, no other changes. |

**Panel (Page Context Panel).** Host gives the panel a fixed narrow column ‚Äî typically **320‚Äì420px**. We design for **360px** as the canonical width; tolerate ¬±60px gracefully.

| Breakpoint (iframe width) | Behavior |
|---------------------------|----------|
| < 320px | Anchor block stacks vertically; coordinate-pair badge wraps under rendering name on each row. |
| 320‚Äì420px | Default. Two-column row layout (rendering name | counter). |
| > 420px | Same; just more breathing room. |

**Drawer (`@blok/sheet`)** is right-anchored; respects the iframe width, never overflows. On the panel surface, the drawer is full-width-of-iframe (the panel is too narrow for a side drawer).

### 3.4 ‚Äî Light / dark mode parity

Both modes inherit from Blok semantic tokens. There is **no** custom palette branch. The widget and panel render correctly under both `<html class="dark">` and the default light mode. Specifically:

- The "isobar separator" reads as `border-border` in both modes ‚Äî a 1px hairline.
- The coordinate-pair badge reads as `bg-background` + `border-border/60` ‚Äî visible in both modes without contrast tweaks.
- The "you are here" primary dot reads as `bg-primary` ‚Äî the Blok primary color is contrast-balanced for both modes.
- The "surveying pulse" uses `--primary` for the moving point and `--ring` at 40% opacity for the trail; both tokens flip automatically.

**Theme source of truth (architecture OQ-5):** the design assumes the Cloud Portal hosts both modes and the iframe inherits via the standard `next-themes`/`.dark` class pattern. Open question deferred ‚Äî see ¬ß 6.

### 4.3 ‚Äî Accessibility notes

#### Keyboard navigation (NFR-4.1)

- **Widget table:** `Tab` enters the table at the first row; `‚Üë/‚Üì` arrow keys move row focus; `Enter` opens the drawer for the focused row; `Esc` closes the drawer and returns focus to the row. The column headers are skipped (sort is not in v1; OS-15).
- **Panel rendering list:** `Tab` enters the list at the first row; `‚Üë/‚Üì` moves; `Enter` activates the coordinate-pair badge ‚Üí opens drawer.
- **Drawer:** `Tab` cycles through the page list; `Enter` triggers the page click-through (`pages.context` mutate); `Esc` closes the drawer and returns focus to the originating row.
- **Search box:** `/` keyboard shortcut focuses the search input from anywhere in the widget (advertised in tooltip).
- **`<LoadingVisualizer>`:** the Cancel button is reachable via `Tab` from any focused element on the surface during scan.

#### Screen reader (NFR-4.2 / ADR-0011)

- `<LoadingVisualizer>` exposes an `aria-live="polite"` region. Announcement contract:
  - On phase transition: *"Phase 2 of 3, mapping pages."*
  - On stall (>5s no progress): *"Slow connection. Try again or cancel."*
  - **Not** announced: per-page progress counter (would be a stream of noise).
- Coordinate-pair badges expose `aria-label`: e.g., `aria-label="12 other pages share this rendering ‚Äî open detail"`.
- The "you are here" anchor exposes `aria-label="Active page: Homepage"` on the primary-color dot.
- The disambiguation suffix `¬∑ ab7c4f1` is hidden from screen readers via `aria-hidden="true"` on the suffix span; the row's overall `aria-label` includes the full ID for SR users.
- Tooltips use the `@blok/tooltip` accessible pattern (Radix-based) ‚Äî keyboard focus reveals; ARIA attributes are wired by the primitive.

#### Color contrast (WCAG AA)

All text on all surfaces hits AA at minimum because Blok's semantic tokens are pre-validated for AA. We do not paint outside Blok tokens, so we inherit the contract. Specific high-attention items:
- Coordinate-pair badge text on `bg-background`: AA ‚â• 4.5:1.
- "Last scanned" text in `text-muted-foreground` on `bg-muted/40`: AA ‚â• 4.5:1 (verified on Blok's neutral pairings).
- Warning badge color: AA via the warning text token, not a custom hex.
- Primary-color dot on `bg-muted/40` (anchor block): the dot is decorative; the page name carries the real signal.

#### No color-only signals (NFR-4.3)

- Stale timestamp uses both color (warning text token) AND a `‚öÝ` icon AND the word "Stale?".
- Missing datasource uses both color (warning) AND `alert-triangle` icon AND the word "Missing".
- Active scan uses both color (primary on the moving point) AND animation AND the phase label.
- Skipped pages use both a chip color AND a reason word (`forbidden` / `timeout` / etc.).

#### Focus management

- Opening the drawer moves focus to the first focusable element inside (the rendering name header).
- Closing the drawer returns focus to the originating row.
- The scan starting moves focus to the cancel button (after a 500ms delay so it doesn't feel jumpy).

#### RTL

Blok ships RTL support via logical properties (`ms-*` / `me-*`). The design uses **only** logical Tailwind properties ‚Äî no `ml-*` / `mr-*`. The coordinate-pair badge's `arrow-up-right` icon flips to `arrow-up-left` in RTL via a CSS `:dir(rtl)` selector that swaps the icon name (engineer detail).

## 5. Responsiveness and accessibility

### 5.1 ‚Äî Breakpoints (consolidated)

The Blok theme exposes default Tailwind breakpoints (`sm 640px`, `md 768px`, `lg 1024px`, `xl 1280px`, `2xl 1536px`). The Atlas overrides none of them ‚Äî but the iframe-bound surfaces practically only see a subset.

| Surface | Effective range | Practical breakpoints |
|---------|-----------------|------------------------|
| Widget | 720‚Äì1440px+ | At `lg` (‚â•1024px): drawer expands to 560px; at `xl` (‚â•1280px): drawer at 640px. Below `md` (theoretical): single-column "compact" mode. |
| Panel | 320‚Äì420px (typical 360px) | Below 320px: stack rows. Above 420px: same layout, more spacing. |
| Drawer | Fits its iframe | Widget surface: side-anchored 480/560/640px. Panel surface: full-width-of-iframe overlay. |

### 5.2 ‚Äî A11y requirements (consolidated)

| Requirement | How met |
|-------------|---------|
| **WCAG 2.1 AA** | Inherited from Blok semantic tokens; no custom palette. |
| **Keyboard navigation everywhere** | Per ¬ß 4.3. Tab order is logical: ribbon ‚Üí search ‚Üí table/list ‚Üí drawer. |
| **`aria-live` for scan progress** | `<LoadingVisualizer>` polite region; phase transitions only (not per-page noise). |
| **No color-only signals** | All status uses icon + text. |
| **Focus visibility** | Blok focus ring (`outline outline-2 outline-ring outline-offset-2`) on every interactive element. |
| **RTL** | Logical Tailwind properties (`ms-*`, `me-*`) only; icon directions flip via `:dir(rtl)`. |
| **Touch targets** | Coordinate-pair badge minimum hit area `32√ó32px` (Blok button-sm baseline); drawer page rows at least 40px high. |
| **Reduced motion** | The "surveying pulse" respects `@media (prefers-reduced-motion: reduce)` ‚Äî animation collapses to a static phase label + counter; trail is hidden. Documented as DoD-6 supplement. |

### 5.3 ‚Äî Performance

- 60fps animations via CSS `transform`/`opacity` only (per ADR-0011 / AR-8).
- Table virtualization at >200 rows (gap noted in ¬ß 6 OQ-V1-1).
- `useMemo` on the disambiguation map (per architecture AR-3).
- No re-render storms on the panel's per-page swap (per AR-5 ‚Äî separate React subtree).
- No layout thrash; no `box-shadow` animation; no opacity on full-surface backgrounds (would force composite-layer regen).

### 6.2 ‚Äî Tensions / risks against architecture or PRD constraints

The variant is **architecturally clean** ‚Äî no part of the cartographic direction fights the locked decisions. Specific items I checked:

| Constraint | Variant 1 status |
|------------|------------------|
| ADR-0009 ‚Äî Blok-only UI, semantic tokens, no hex | Honored. Every component named is `@blok/*` or composed from Blok primitives + semantic tokens. The cartographic feel comes from layout + typography + motion, not from custom palette. |
| ADR-0011 ‚Äî branded animation, no mini-game, no generative bloom | Honored. The "surveying pulse" is a single-mode branded animation; no "Distract me" tri-state; no canvas-based bloom; no mini-game. |
| ADR-0004 ‚Äî two iframe entries, one shared atlas | Honored. The widget and panel share `<UsageDrawer>`, `<LoadingVisualizer>`, `<DirectBindingsAffordance>`, `<CoordinatePairBadge>`, `<RenderingNameCell>` ‚Äî primitive set is identical across surfaces. |
| ADR-0005 ‚Äî rendering = definition ID, datasource = item ID; pages-not-items | Honored. Item IDs only appear in tooltips, copyable badges, and click-through hrefs. Display names lead. Disambiguation suffix is the sole exception, surfaced as a tiny inline `@blok/badge`. |
| ADR-0006 ‚Äî direct datasource bindings only + mandatory `‚ìò` affordance | Honored. `<DirectBindingsAffordance />` is non-dismissible and visible on every surface where datasource counts appear (widget freshness ribbon + panel Datasource impact section). |
| ADR-0010 ‚Äî module singleton + `useSyncExternalStore` | Design-neutral, but the freshness ribbon and counters are designed to subscribe to fine-grained slices (timestamp, totals, per-rendering-count) so the per-tick re-render cost is bounded. |
| Session-scoped cache (FR-4) | Honored. Freshness ribbon is the only place the cache is exposed; "Refresh atlas" is the only invalidation control surfaced to the editor. |
| WCAG AA | Honored ‚Äî see ¬ß 4.3 / ¬ß 5.2. |
| Light + dark parity | Honored ‚Äî pure semantic-token use; no palette branch. |

**Mild tension worth flagging:**
- The cartographic metaphor risks **overpromising spatial relationships** that don't actually exist in the data. Atlas does not show *where* on a page a rendering sits; it shows *whether* and *how many*. The visual language must avoid suggesting layout-on-page meaning. The mitigation here is restraint ‚Äî no map graphics, no x/y coordinates, no visualizations of placeholder structure. Just the chart-legend aesthetic + coordinate-pair counter look. If user testing finds editors expect "show me where on the page" from this metaphor, variant 2 or 3 may serve them better.
- The "surveying pulse" three-icon trace must be readable in the panel's narrow 320‚Äì420px width. The trace path is short; design assumes ~200px between site/page/component icons. Verify on a 320px iframe on first scaffold.

### 6.3 ‚Äî What I deliberately did NOT do

- **No literal map graphics.** No SVG of continents, no globe icon as the headline, no "geographic" colors. The cartographic feel is pure typographic + layout discipline.

- **No dark-mode-only flourishes.** Both modes get the same visual hierarchy.
- **No "Distract me" toggle.** Removed per ADR-0011.
- **No mini-game.** Per OS-12 / ADR-0011.
- **No persisted preferences UI.** Session-scoped only per FR-4.
- **No URL-construction click-through.** SDK `pages.context` mutate only.
